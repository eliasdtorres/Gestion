---------- Sistema de control de ventas y caja-------------------

El proyecto consta del desarrollo de una app bajo el lenguaje Java usando MySql para la recoleccion de los datos, Spring Boot y Spring MVC y Spring Security, para el front se pretende utilizar además de HTML y CSS, Thymeleaf/Angular.

La aplicación se trata de un control de ventas y cierre de caja. En dicha aplicación se debe poder cargar articulos con las siguientes caracteristicas:

  • - id de articulo
  • - codigo de artículo
  • - nombre de artículo
  • - stock de existencias
    • se debe actualizar según venta o ingreso de mercadería
  • - precio de venta de artículo
  • - costo de articulo
  • - cantidad vendida
  • - está en promoción?
    • si está en promociónn debe indicarse en pesos y porcentaje
  • - selector de tipo : venta/cambio/cambio+venta
  • - campo vacío para agregar descuentos adicionales (en pesos) o quitar descuento según se requiera
    • debe poder visualizarse en tabla como porcentaje y pesos
  • - diferencia venta/costo
    • debe poder visualizarse en tabla como porcentaje y pesos

    • Avanzado

    • - generador de numero de ticket para cambios
      • debe indicar codigo de articulo
    • El proyecto debe contar con BBDD
      • - donde se aloje el stock
      • - las ventas diarias
        • las tablas de venta diaria deben poder agregar gastos de caja (compras de insumos u otros)
      • - el cierre de caja
        • las tablas del cierre de caja deben poder ser editables desde la aplicaciónn
    • - las tablas pueden ser editadas
